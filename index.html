Sass (Syntactically Awesome StyleSheets)

Hojas de estilo con superpoderes

Sass es una extensión de css que agrega capacidad al lenguaje de utilizar herramientas que aún o están soportadas en el leguaje

Por ejemplo: Variables, reglas anidadas, mixins(como funciones en css), importar reglas de estilos "en linea"

Para utilizar sass podemos utilizar dos tipos de sintaxis: SCSS (Sassy CSS) y SASS

SCSS: (Sassy CSS)

Esta sintaxis es sumamente parecida a la de css sin sass, una de sus mayores ventajas es que es reconocida por todos los navegadores, incluido internet explorer en antiguos navegagodes. En teoría todo archivo de css puede ser potencialmente un archivo de scss.

¿Cual es su sintaxis?

Identificamos la sintaxis de un archivo scss pues las reglas de estilo están agrupadas entre llaves

.container{
__color:"black";
__backgroud-color:"white";
}
Sass

La sintaxis de sass permite trabajar las reglas de estilo a través de indentación.

¿Cual es su sintaxis?
.container
__color:"black"
__backgroud-color:"white"

Command Line

Configurando un archivo de sass

  1. En tu command prompt navega hasta la carpeta donde se encuentra tu proyecto (usa el comando cd)
  2. En la carpeta de tu proyecto build crea un archivo con extensión .scss (touch main.scss)
  3. Este archivo será el que sea preprocesado con SASS
  4. Ahora vamos a observar los cambios que se hagan en ese archivo principal
    5. sass --watch archivo_scss_o_sass_fuente.scss:archivo_destino_css_a_produccion.css sass --watch css/main.scss:production/css/main.css al hacer esto estamos creando un nuevo archivo main.css en la carpeta de producción de nuestro proyecto
Ahora, podemos comenzar a escribir con sass los estilos de nuestro proyecto.

Variables

Las variables en SASS son una forma de almacenar valores de CSS que te gustaría reutilizar en algún momento. Puede ser cualquier cosa: colores, fuentes, valores predeterminados de pocisión, etc.

Sintaxis de SCSS


__$main-font: Helvetica, sans-serif;
__$primary-color: #333;
__body {
____font: 100% $main-font;
____color: $primary-color;
__}
¿Por qué esto te facilita la vida?

Que pasa si el equipo de diseño decide que es un buen momento para modificar TODOS LOS COLORES del producto. Sin un preprocesador de css tendrías que moverte por cada linea de tus paginas de estilos para modificar a mano cada lugar donde los colores fueron utilizados.

Al tener variables para los colores, basta con modificar el color en la declaración de la varibale y volver a Construir tu archivo main.css para que todos los colores del documento que hagan uso de ese valor se actualicen

Sintaxis de SCSS


__$main-color: #fff;
__$secondary-color: #333;
__$emphasis-color: #d0398a;
__main{
____color: $main-color;
____background-color: $secondary-color;
__}

Anidamiento || Nesting

Sass permite anidar selectores en css de la misma forma jerarquica que lo hace HTML. Pero ten cuidado, con este gran poder, viene una gran responsabilidad. Debes tener cuidado de no sobreanidar elementos en tu CSS para no hacer complicada tu hoja de estilos de mantener (sobreanidar puede ser una mala práctica)

¿Por qué la propiedad de Nesting te facilita la vida?
  • Simplifica las reglas de especificidad de CSS
  • Simplifica la modificación de elementos hijo dentro de elementos padres.
main{
__ color: $emphasis-color-one;
__ background-color: $main-color;
__ h1{
____color:$emphasis-color-two;
__ }
__p{
____color: $secondary-color;
__ }
__code{
____color: $emphasis-color-two;
__ }
}

Partials || Archivos Parciales

Pero Meme, que pasa si tengo 50 variables y aparte 50 mixins y resulta que son para cosas distintas. Existe alguna forma de Modularizar mis hojas de CSS con SASS

Sí, así es. Puedes crear un archivo parcial para "hacer módulos" de tus hojas de estilo

Lo unico que Sass te pide es que los nombre con un guión bajo al inicio, algo así: _colors.scss

y después lo importes al archivo principal de tus estilos (en este caso main.scss) de la siguiente forma @import "colors";

¿@import? Okay, eso luce extraño, explicame que es, por favor :) .

Imports

Como decía anteriormente, sass permite dividir tu código de css en pequeñas porciones. lo que hace mucho más mantenible los estilos de tu sitio.
Imagina que tienes una landing page en donde tienes 5 secciones, un footer y un header Te piden que modifiques solo el header pero ¡oh sorpresa! tu código de CSS, tiene más de 1500 líneas y la persona desarrolladora que estaba antes de ti era muy desordenada. ¡Caos!
Si tuvieras tu hoja de estilos dividida en archivos parciales, sería mucho más sencillo ir al archivo "_header.scss" y darle mantenimiento a ese pequeño pedazo de estilos, sin tanto peligro de romperlo todo.

Una vez modificados, solo tienes que importar ese archivo parcial en tu hoja de estilos base a través de la palabra reservada @import seguida del nombre del archivo parcial, sin el guíon bajo y sin extensión 'header'.

Sass es suficientemente inteligente para reconocer que ese será un archivo parcial y llevar esos estilos a la hoja principal. ¡No lo olvides!
@import 'nombre-cool-archivo-parcial';
al inicio de tu archivo principal main.scss

Mixin

¡Mi parte favorita ha llegado! ¡Con sass no hay excusa para tener código repetido incluso en CSS! DRY personas, ¡DRY Everywhere!

Los MIXIN en sass son similares a las funciones que utilizamos en nuestros lenguajes de programación preferidos. Como bien sabes, las funciones están hechas para agrupar un proceso y obtener un resultado las veces que sean necesarias. Cuando utilizamos ua función evitamos la penosa necesidad de escribir el mismo código muchas veces y nos limitamos a hacer una llamada a la función deseada en el momento deseado.

Un mixin te permite hacer grupos de declaraciones en CSS que puedes reutilizar en varias partes de tu sitio. Incluso te permite pasar valores al Mixin (como los argumentos de una función), para hacer al mixin más flexible.

Hagamos un mixin simple que modifique la fuente, segun el argumento que le enviemos (Tal vez este no es el ejemplo más cool de todos pero cuando veamos operadores, podremos hacer uno mucho más completo)

__@mixin fuente($value) {
____font-family: $value;
__}

Para hacer uso de nuestro mixin debemos incluirlo en la regla que deseemos

__code{
____ @include fuente("monospace");
__}
