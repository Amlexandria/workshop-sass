Sass (Syntactically Awesome StyleSheets)

Hojas de estilo con superpoderes

Sass es una extensión de css que agrega capacidad al lenguaje de utilizar herramientas que aún o están soportadas en el leguaje

Por ejemplo: Variables, reglas anidadas, mixins(como funciones en css), importar reglas de estilos "en linea"

Para utilizar sass podemos utilizar dos tipos de sintaxis: SCSS (Sassy CSS) y SASS

SCSS: (Sassy CSS)

Esta sintaxis es sumamente parecida a la de css sin sass, una de sus mayores ventajas es que es reconocida por todos los navegadores, incluido internet explorer en antiguos navegagodes. En teoría todo archivo de css puede ser potencialmente un archivo de scss.

¿Cual es su sintaxis?

Identificamos la sintaxis de un archivo scss pues las reglas de estilo están agrupadas entre llaves

.container{
__color:"black";
__backgroud-color:"white";
}
Sass

La sintaxis de sass permite trabajar las reglas de estilo a través de indentación.

¿Cual es su sintaxis?
.container
__color:"black"
__backgroud-color:"white"

Instalación

Windows

En primer lugar debes recordar que sass es una gema de Ruby (un componente de ese lenguaje) Por tanto, antes que hacer cualquier cosa, debemos instalar Ruby en nuestros equipos.

  • Instalador de Ruby en Windows
  • Tutorial de instalación de Ruby en Windows

Una vez que tengas instalado Ruby, podremos instalar SASS
gem install sass Si tu consola te da algún error por permisos intenta de esta forma:
sudo gem install sass

Si no se desea utilizar Sass como una gema de Ruby, entonces debe configurarse como una variable de entorno en tu sistema operativo e instalarse a través del package manager de tu preferencia(npm, choco, brew) aquí un tutorial para hacerlo de esta forma: Configurar algo como variable de entorno Install Sass (oficial doc latest version) 2. Command Line

Configurando un archivo de sass

  1. En tu command prompt navega hasta la carpeta donde se encuentra tu proyecto (usa el comando cd)
  2. En la carpeta de tu proyecto build crea un archivo con extensión .scss (touch main.scss)
  3. Este archivo será el que sea preprocesado con SASS
  4. Ahora vamos a observar los cambios que se hagan en ese archivo principal
    5. sass --watch archivo_scss_o_sass_fuente.scss:archivo_destino_css_a_produccion.css sass --watch css/main.scss:production/css/main.css al hacer esto estamos creando un nuevo archivo main.css en la carpeta de producción de nuestro proyecto
Ahora, podemos comenzar a escribir con sass los estilos de nuestro proyecto.

Variables


Las variables en SASS son una forma de almacenar valores de CSS que te gustaría reutilizar en algún momento. Puede ser cualquier cosa: colores, fuentes, valores predeterminados de pocisión, etc.

Sintaxis de SCSS


__$main-font: Helvetica, sans-serif;
__$primary-color: #333;
__body {
____font: 100% $main-font;
____color: $primary-color;
__}
¿Por qué esto te facilita la vida?

Que pasa si el equipo de diseño decide que es un buen momento para modificar TODOS LOS COLORES del producto. Sin un preprocesador de css tendrías que moverte por cada linea de tus paginas de estilos para modificar a mano cada lugar donde los colores fueron utilizados.

Al tener variables para los colores, basta con modificar el color en la declaración de la varibale y volver a Construir tu archivo main.css para que todos los colores del documento que hagan uso de ese valor se actualicen

Sintaxis de SCSS


__$main-color: #fff;
__$secondary-color: #333;
__$emphasis-color: #d0398a;
__main{
____color: $main-color;
____background-color: $secondary-color;
__}
